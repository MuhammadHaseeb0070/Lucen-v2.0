# Phase 16-01: Dedicated Patch System Prompt & Sidecar Call

## Objective
Create a dedicated system prompt for patching and execute it via a sidecar call.

## Must Haves
- Server-side Deno Edge Function `chat-proxy` must support `patch: true` boolean in body.
- When `patch: true` is passed, `chat-proxy` forces `stream: false`, `call_kind: 'patch'`, and applies/verifies `PATCH_SIDECAR_SYSTEM_PROMPT` as the system instruction.
- A client-side service `src/services/openrouter/patchClient.ts` performs the non-streaming POST request to `chat-proxy`.
- Parser checks for `FULL_REGEN_REQUIRED` and `AMBIGUOUS_PATCH` sentinels and fallback automatically to standard full regeneration.
- Successfully applied patches are stored in conversation history (content contains only the patch block) with the `isPatch: true` metadata flag.
