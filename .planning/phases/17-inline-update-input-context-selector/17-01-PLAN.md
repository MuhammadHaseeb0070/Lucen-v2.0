# Phase 17-01: Inline Update Input & Context Selector

## Objective
Build the inline update input box and context selector below artifacts with a premium, glassmorphic layout that integrates statically at the bottom of the workspace to avoid overlaps.

## Must Haves
- Redesign `ArtifactPatchInput.tsx` completely from scratch with the premium glassmorphic UI.
- Implement the Segmented Context Pill Selector next to the input, offering options `0`, `1`, `2`, `3` messages (represented as pills). Default to `0`.
- Update `ArtifactWorkspace.tsx` to mount `ArtifactPatchInput` inside the flex vertical layout below the code/render pane, replacing the absolute positioning to prevent overlap.
- Support the inline pipeline progress states, displaying the corresponding label directly inside the input container during patch execution.
- Support failure and success visual states (red flash on error, green flash on completion).
