# Phase 15: Patch Format & Parser Migration

## Objective
Update the artifact patch parser to understand Git conflict markers instead of XML tags, with lenient matching to protect against LLM syntax variations. Add post-patch HTML sanity checking via `DOMParser`, and surface sentinels through a unified status object.

## Must Haves
- The parser must correctly parse `<<<<<<< SEARCH`, `=======`, and `>>>>>>> REPLACE` markers, even with trailing whitespace or optional labels (e.g. `<<<<<<< SEARCH HTML`).
- The parser must surface `FULL_REGEN_REQUIRED` and `AMBIGUOUS_PATCH` sentinels as a structured status object.
- The patch engine must run a post-patch `DOMParser` check for `html` and `svg` artifacts, failing the patch if a `<parsererror>` node is detected.
